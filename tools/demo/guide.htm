Introduction

VoltDB is a visionary "NewSQL" platform: the promise to deliver outstanding data access performance for demanding Big Data application while scaling on commodity hardware.
  • Unlike traditional databases bottlenecked by disk I/O, VoltDB can execute millions of transactions per second!
  • Unlike "NoSQL" solutions, you work with a relational model and avoid data corruption with transactional/concurrency control.
The following samples show you some uses cases where VoltDB truly excels and achieves the impossible...

'American Idol' Contest ('Voter')

Imagine a competition for a prime-time TV show.
  • 6 artists
  • Millions of viewers
  • 2 minutes to vote for your favorite
With millions of callers hitting the phone, how do you build a system that tracks votes but prevents fraud?
  • Viewers can only vote for the same contestant once
  • Viewers are allowed a maximum of 3 votes
  • Report the final winner in less than 1 minute.
Running the sample
In a Terminal, run:
launch voter
This will open two Terminal windows:
  • A VoltDB database process.
  • A call center client capturing phone calls.
Next, click here to launch VoltDB Studio.
What is happening?
  • The call center is randomly posting votes.
  • VoltDB validates each vote, blocking fraud.

The charts in VoltDB Studio (right side) show how many votes the database is processing ('Transactions/s') and how long it takes to process each vote ('Latency (ms)').

The obvious goal: high Transaction/s, low latency (more votes, faster!)

At the end of the benchmark, the call center simulator reports the contest winner (in the Terminal window), a query that takes a mere fraction of a second, despite millions of votes having been processed!

Explore...

You can open a New Query in VoltDB Studio to run requests against the database even as the application is running.

Follow live voting results:

Get the total number of votes:
select count(*) from votes;

Key/Value Store

VoltDB is far more versatile than what 'NoSQL' systems, but if all you need is a high-speed Key/Value store this sample shows you how.
  • String keys
  • Binary values up to 1MB
Binary storage lets you use any logic you might want on the client side: you could for instance:
  • Serialize client object in binary form
  • Store zipped content/files
  • Store Unicode string data
Running the sample
In a Terminal, run:
launch key_value
This will open two Terminal windows:
  • A VoltDB database process.
  • A client benchmark application.
Next, click here to launch VoltDB Studio.
What is happening?
  • The client runs various PUT/GET ops (12K payload).
  • VoltDB serves as a simple high-speed data store.

The charts in VoltDB Studio (right side) show how many votes the database is processing ('Transactions/s') and how long it takes to process each operation ('Latency (ms)').

The obvious goal: high Transaction/s, low latency (more votes, faster!)

At the end of the benchmark, the client application reports performance results (in the Terminal window)

Explore...

You can open a New Query in VoltDB Studio to run requests against the database even as the application is running.

Sample out store data:
select top 10 * from key_value;
Get the total number of stored values:
select count(*) from key_value;
